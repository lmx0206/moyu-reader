## 🔴 Critical（必须修）

### C1 — EPUB href 未百分号解码，章节被静默丢弃
- **位置**：`internal/book/epub/opf.go:54`（`joinArchive(opfDir, href)`，只做 `path.Clean`），`internal/book/epub/epub.go:45-49`（`files[href]` 精确匹配 zip 名，失败后第 48 行 `continue`）。
- **问题**：真实 EPUB 的 OPF 里 href 常是百分号编码，如 `Text/chapter%201.xhtml` 或非 ASCII 文件名编码形式；而 zip 条目名是**解码后**的形态。`files[href]` 命中失败 → 第 48 行**静默 `continue`** → 书「导入成功」但缺章，无任何报错。
- **为何重要**：这是真实世界 epub 最可能踩到的导入失败，且无声无息——用户只会觉得「这本书怎么少了几章」。
- **修复**：在 `joinArchive` 之前对 href 做 `url.PathUnescape`（失败则回退原值）；建议再加一个大小写规范化的兜底匹配。`epub.go:48` 的「跳过不可读章节」逻辑可保留，但应在跳过时累计计数，全书 0 章时返回明确错误。
- **测试缺口**：`epub_test.go` 只用纯 ASCII href，需新增「百分号编码 / 非 ASCII href」夹具，断言章节不丢。

### C2 — 全局鼠标捕获破坏复制粘贴
- **位置**：`internal/ui/run.go:13`（`tea.NewProgram(m, tea.WithAltScreen(), tea.WithMouseCellMotion())`）。
- **问题**：`WithMouseCellMotion()` 是**全局**开启的，但鼠标事件唯一用途是 REPL 滚轮滚动（`model.go:140-150` 的 `tea.MouseMsg` 分支）。开启后终端无法用鼠标拖选/复制文本（得到的是 xterm 鼠标转义序列而非选区），且在书架/阅读等无鼠标逻辑的界面也照样捕获。
- **为何重要**：对一个「装作在看日志」的工具，不能拖选「日志输出」既可疑又恼人，直接削弱伪装前提。`cell-motion`（移动追踪）尤其过度——没有任何代码读取移动事件。
- **修复**：去掉全局鼠标捕获；若要保留 REPL 滚轮，改为**仅在进入 REPL 时动态开启、离开时关闭**（bubbletea 支持把 `tea.EnableMouseCellMotion`/`tea.DisableMouse` 作为 `tea.Cmd` 返回来动态切换）；或干脆只靠 PgUp/PgDn 滚动、彻底移除鼠标。优先级：动态开关 > 仅 PgUp/PgDn > 现状。
- **背景**：此项是 v0.6.1 加滚轮时引入的已知取舍（见记忆 `moyu-reader-project.md` v0.6.1 条），现确认需改进。

## 🟠 Important（应修）

### I1 — 非阅读界面明文中文，伪装破功
- **位置**：
  - 导入屏：`internal/ui/model.go:532` —「导入 EPUB（粘贴 .epub 完整路径后回车，Esc 取消）」
  - 加标注屏：`model.go:538` —「加标注（批注可留空 = 书签，回车保存，Esc 取消）」
  - 报错 status：`model.go:87`「找不到这本书」、`:94`「这本书打不开（已标记损坏）」、`:499`「解析失败: 」+ 原始错误
  - 书架：`internal/ui/shelf.go:63` —「📚 书架   ↑↓ 选择 · Enter 阅读 · i 导入 · d 删除 · q 退出」、`:58`「书架空空如也。」
- **问题**：这些屏幕不经 `paintShell`/`paintDim`，无任何假工具框架，是赤裸的中文「小说阅读器」字样；那个 `📚` emoji 尤其扎眼。一离开阅读视图，伪装瞬间穿帮。
- **为何重要**：项目的全部价值在伪装；阅读视图做得很好，但 shelf/import/annotate/error 是同等暴露面。
- **修复**：把书架/导入/加标注/帮助/报错都走伪装层（或至少中性的假 CLI 框架，如假 `git commit -m` 提示、文件路径提示）；移除明文中文与 emoji。注意把原始解析错误（`model.go:499` 直接拼 `err.Error()`，如「epub: required archive entry missing」）也包装成中性 CLI 风格。
- **测试缺口**：加断言「`View()` 在 import/annotate/shelf/error 屏不含明文中文 UI 字样」。

### I2 — REPL 阅读模式不会被记住
- **位置**：`internal/ui/model.go:385-395`（`cycleMode`）、`:404-412`（`handleReplKey` 的 shell 回切）。
- **问题**：`cycleMode` 在 `Prefs().Mode == "inline"` 时创建 `ReplView`，但**从不把 reader 的 mode 置为 "repl"**。`saveProgress` 持久化的是 `m.reader.Prefs()`（仍是 inline）。`ReplView.Prefs()` 虽返回 `Mode:"repl"` 但退出时从未被存。结果：重开书永远回到 shell/inline，第三种模式无法恢复。
- **为何重要**：三种阅读模式里只有 REPL 不被记忆，是功能不一致。
- **修复**：引入 `mode=="repl"` 的持久化往返——退出 REPL 到书架/退出程序时，存 `ReplView.Prefs()`（或在 reader 上记录 repl 标记）；`openBook` 时若 `Prefs().Mode=="repl"` 则直接进 REPL。
- **测试缺口**：`repl_test.go` 只覆盖会话内行为，需加「REPL → 退出 → 重开仍是 REPL」往返测试。

### I3 — 阅读时长统计不准
- **位置**：`internal/ui/model.go:337`（`handleReaderKey` 入口 `recordActivity`）、`:398`（`handleReplKey` 入口）、`:103`（`openBook` 才重置 `lastActivity`）、`:562`（5 分钟 gap 上限）。
- **问题**：本质是「相邻按键的时间间隔之和」，不是真实阅读时间：
  - 静默读完一长页再翻页 → 整段思考时间（≤5 分钟）全计入；
  - 在 TOC/stats/help/annotate 屏停留不重置时钟，回到 reader 的下一次按键会把这段间隔计入；
  - 切书中途不重置 `lastActivity`，跨书累计。
- **为何重要**：stats 面板的「今日/累计时长」会偏大、显得不准；streak 本身正确。
- **修复**：方案可选——(a) 把计时改为「停留即计」的心跳（如基于 bossTick 类似的定时器，只在 reader/repl 活跃屏累加固定步长）；(b) 至少在切屏（进入 TOC/stats/help/annotate/boss）和切书时重置/暂停 `lastActivity`，并缩小 gap 上限。先记录，修复时定方案。
- **测试缺口**：跨屏、跨书、长 gap 的计时断言。

### I4 — 每次按键全章重复 re-wrap
- **位置**：`internal/ui/reader.go:145-152`（`chapterLines()` 调 `LayoutChapter` 全章重排），被 `Render`/`PageDown`/`PageUp`/`LineDown`/`LineUp`/`StatusText`/`Progress`/`SetSize` 多处调用；`Progress`/`SetSize` 还另外调 `ParagraphStartLines`（`render/anchor.go:14`）再 wrap 一次。
- **问题**：长章节（数万 runes）每次方向键都对整章重排多次。慢机器上有可感知卡顿——一个「卡顿的日志查看器」本身就奇怪。
- **修复**：在 `ReaderView` 加按 `(chapter, contentWidth)` 的排版缓存（行切片 + 段落起始行表），宽度或章节变化才重算。注意与 `SetSize` 重锚逻辑配合。
- **测试缺口**：可加基准/行为测试确保缓存命中后输出不变。

### I5 — 窄终端下 shell 进度状态串被截掉
- **位置**：`internal/disguise/theme.go:81-83`（`Footer` 把阅读状态嵌进 `left`）、`internal/disguise/util.go:39-42`（`padBetween` 空间不足时截 `left` 保 `right` 标记）。
- **问题**：窄终端下 `· 0%` 进度（shell 模式唯一的章内位置提示）会消失且无兜底。
- **为何重要**：shell 模式用户唯一的进度线索。功能性轻，但属可见缺陷。
- **修复**：窄宽时优先保留进度百分比（而非 `? help` 标记），或为状态串设最小可见预算。
- **测试缺口**：`padBetween` 窄宽截断行为测试。

## 功能 / UX 缺口（用户实际会遇到，按优先级）

1. 真实 EPUB（编码 href）→ 缺章无提示（C1）——最高影响。
2. 不能鼠标拖选复制（C2）——且全界面开启。
3. 书架/导入/加标注屏明文「书架/导入 EPUB/加标注」（I1）——离开阅读视图即破功。
4. REPL 模式不被记忆（I2）——重开总回 shell/inline。
5. 非 epub 的 zip（缺 `container.xml`）→ 生硬中文报错 `"解析失败: epub: required archive entry missing"` 直接上屏（I1 关联，`model.go:499`）。
6. 超长单段：`PageUp` 跨章回看可能落在段中，因 `lastPageStart` 基于行（`reader.go:211-216`）而位置基于段；经 `Progress()` 重入会吸附到段首，上一章「最后一页」可能与刚才所见不同——轻微回流意外。
7. REPL 中改变窗口大小：scrollback 保留原文（`repl.go:54-56`），旧行按旧宽度 wrap、不重排，缩放后回看可能参差。可接受但可见。
8. stats 数字偏近似（I3）：streak 正确，「今日/累计秒数」是按键间隔和。
9. 无窗口标题管理：altscreen 不恢复自定义标题；exe 名即标签名。低优先。
